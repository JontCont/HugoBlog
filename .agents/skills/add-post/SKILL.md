# Skill: 撰寫技術部落格文章

## 說明

根據本部落格的寫作慣例，協助建立格式正確、結構清晰的 Hugo 技術文章。
輸出包含完整 TOML frontmatter 與主體內容，可直接放入 `content/posts/` 使用。

---

## 適用時機

- 撰寫新的技術筆記或教學文章
- 整理某主題的使用方式、步驟說明
- 紀錄工具安裝、設定流程
- 心得、比較類文章

---

## 文章撰寫流程

### Step 1 — 確認基本資訊

開始前先確認以下欄位：

| 欄位 | 說明 | 範例 |
|------|------|------|
| `title` | 文章標題（含分類標記） | `'【C#】- SignalR 使用方式'` |
| `date` | 發布日期時間 | `'2024-07-21T11:51:23'` |
| `slug` | URL 識別名（與檔名日期後半段一致） | `'CSharp-SignalR-使用方式'` |
| `categories` | 分類（1~2個） | `['後端技術', 'C#']` |
| `tags` | 標籤 | `['C#', 'SignalR']` |
| `keywords` | SEO 關鍵字 | `['SignalR', 'WebSocket']` |
| `image` | 封面圖片路徑 | `'/image/20240721_cover.png'` |

### Step 2 — 決定文章結構

依文章類型選擇適合的大綱：

**教學 / 操作型（最常見）**
```
前言
一、介紹 / 說明
二、環境 / 必備工具
三、步驟說明（可拆多節）
四、範例程式碼
結論（可選）
```

**介紹 / 概念型**
```
前言（或直接開頭說明）
主概念介紹（用 ## 分節）
各子項說明
小結 / 結論（可選）
```

**心得 / 筆記型**
```
前言
主要內容（自由段落）
結論（可選）
```

### Step 3 — 撰寫內容

遵守以下格式規則：

1. **標題層級**：大節用 `##`，子節用 `###`，再細分用 `####`
2. **章節編號**：教學類用中文數字 `一、二、三…` 搭配步驟編號 `1. 2. 3.`
3. **分隔線**：`---` 用於主要段落之間，增加閱讀節奏
4. **程式碼區塊**：一律標註語言，例如 ` ```cs ``` `、` ```console ``` `、` ```html ``` `
5. **圖片**：使用 `![說明文字](/image/檔名.png)` 或 `![](/image/檔名.png)`
6. **連結**：`[顯示文字](URL)`
7. **行內程式碼**：用反引號 ` `` ` 標記指令、參數名稱
8. **語氣**：口語化但不失精確；可適量使用 emoji 增加親切感

### Step 4 — 命名與存放

檔案命名規則：
```
YYYY-MM-DD-{分類}-{主題}.md
```

範例：
- `2024-07-21-Playwright-使用方式-一-.md`
- `2023-03-02-Git-何謂-Git-Flow.md`
- `2022-07-19-CSharp-SignalR-1-SignalR-使用方式.md`

存放路徑：`content/posts/`

## 品質檢查清單

完成後確認以下項目：

- [ ] frontmatter 所有欄位填寫完整（`description` 可留空、`url` 有無均可）
- [ ] `title` 使用標準格式（`【分類】- 主題` 或 `分類 主題 (N) - 子標題`）
- [ ] 至少有一個 `## 前言` 段落或等效的開場說明
- [ ] 程式碼區塊均有語言標記
- [ ] 圖片路徑以 `/image/` 或 `/img/` 開頭
- [ ] 步驟敘述清楚、可複現
